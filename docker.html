UNIT I: Introduction to Docker

üê≥ What is Docker?

Docker is a containerization platform that packages applications and their dependencies into lightweight, portable containers.

Docker Architecture:

  • Docker Client: CLI interface to interact with Docker
  • Docker Daemon: Background service managing containers
  • Docker Registry: Storage for Docker images (Docker Hub)
  • Docker Objects: Images, Containers, Networks, Volumes

Docker Lifecycle:

1. Build ‚Üí 2. Ship ‚Üí 3. Run

Hyper-V Concept:

Hyper-V is Microsoft's virtualization technology. Docker on Windows uses Hyper-V for container isolation.

üíª Installation & Basic Commands

  1. Download Docker Desktop

  3. Install and restart

Basic Docker Commands:

docker rm container_id # Remove container

‚ùì Viva Questions - Unit I

Q1: What is Docker and why is it used?

Answer: Docker is a containerization platform that packages applications with dependencies into portable containers. It's used for consistent deployment across environments, scalability, and resource efficiency.

Q2: Explain Docker Architecture.

Answer: Docker has 4 main components: Docker Client (CLI), Docker Daemon (background service), Docker Registry (image storage), and Docker Objects (images, containers, networks, volumes).

Q3: What is the difference between Docker Image and Container?

Answer: Image is a read-only template used to create containers. Container is a running instance of an image with its own file system and processes.

Q4: What is Hyper-V and its role in Docker?

Answer: Hyper-V is Microsoft's virtualization technology. Docker on Windows uses Hyper-V for container isolation and virtualization.

Q5: How to check if Docker is running?

Answer: Use commands: docker --version, docker info, or docker ps

üìÑ Dockerfile

Dockerfile is a text file with instructions to build Docker images automatically.

Basic Dockerfile Structure:

FROM ubuntu:20.04 MAINTAINER your-name RUN apt-get update RUN apt-get install -y nginx COPY index.html /var/www/html/ EXPOSE 80 CMD ["nginx", "-g", "daemon off;"]

Dockerfile Instructions:

  • FROM: Base image
  • RUN: Execute commands
  • COPY: Copy files
  • ADD: Copy + extract
  • EXPOSE: Declare ports
  • CMD: Default command
  • ENTRYPOINT: Fixed command
  • ENV: Environment variables
  • WORKDIR: Working directory

Building from Dockerfile:

docker build -t image_name .
docker build -t image_name:tag .

‚ùì Viva Questions - Unit II

Q1: What is Docker Hub?

Answer: Docker Hub is a cloud-based registry service for sharing Docker images. It's the default registry for Docker.

Q2: How to push an image to Docker Hub?

Answer: 1) docker login 2) docker tag image username/repo:tag 3) docker push username/repo:tag

Answer: Dockerfile is a text file containing instructions to build Docker images automatically. It defines the environment and dependencies.

Q4: Difference between CMD and ENTRYPOINT?

Answer: CMD provides default commands that can be overridden. ENTRYPOINT sets fixed commands that cannot be overridden.

Q5: What are Docker volumes?

Answer: Docker volumes are persistent storage mechanisms managed by Docker, independent of container lifecycle.

üêô Docker Compose

Docker Compose is a tool for defining and running multi-container Docker applications using YAML files.

Installation:

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version

YAML File Basics:

YAML (YAML Ain't Markup Language) uses indentation and key-value pairs. No tabs, only spaces!

Sample docker-compose.yml:

version: '3.8' services: web: image: nginx ports: - "8080:80" database: image: mysql:5.7 environment: MYSQL_ROOT_PASSWORD: password volumes: - db_data:/var/lib/mysql volumes: db_data:

WordPress with Docker Compose:

version: '3.8' services: wordpress: image: wordpress:latest ports: - "8080:80" environment: WORDPRESS_DB_HOST: db:3306 WORDPRESS_DB_USER: wordpress WORDPRESS_DB_PASSWORD: wordpress depends_on: - db db: image: mysql:5.7 environment: MYSQL_DATABASE: wordpress MYSQL_USER: wordpress MYSQL_PASSWORD: wordpress MYSQL_ROOT_PASSWORD: rootpassword volumes: - db_data:/var/lib/mysql volumes: db_data:

Docker Compose Commands:

docker-compose up # Start services
docker-compose up -d # Start in background
docker-compose down # Stop services
docker-compose ps # List services
docker-compose logs # View logs
docker-compose build # Build images

üêù Docker Swarm

Docker Swarm is Docker's native clustering and orchestration solution for managing multiple Docker hosts.

Swarm Concepts:

  • Node: Docker engine in swarm
  • Manager Node: Controls swarm
  • Worker Node: Runs tasks
  • Service: Definition of tasks
  • Task: Running container

Creating Docker Swarm:

docker swarm init # Initialize swarm
docker swarm init --advertise-addr IP_ADDRESS
docker swarm join-token worker # Get join token
docker swarm join-token manager

Joining Swarm:

docker swarm join --token TOKEN MANAGER_IP:2377

Managing Nodes:

docker node ls # List nodes
docker node inspect NODE_ID
docker node rm NODE_ID
docker node promote NODE_ID # Make manager
docker node demote NODE_ID # Make worker

Creating Services:

docker service create --name web nginx
docker service create --name web --replicas 3 nginx
docker service create --name web -p 8080:80 nginx

Managing Services:

docker service ls # List services
docker service ps SERVICE_NAME # Service tasks
docker service scale SERVICE_NAME=5 # Scale service
docker service update SERVICE_NAME # Update service
docker service rm SERVICE_NAME # Remove service

‚ùì Viva Questions - Unit IV

Q1: What is Container Orchestration?

Answer: Container orchestration is automated management of containerized applications including deployment, scaling, networking, and health monitoring.

Q2: What is Docker Swarm?

Answer: Docker Swarm is Docker's native clustering and orchestration tool for managing multiple Docker hosts as a single virtual system.

Q3: Difference between Manager and Worker nodes?

Answer: Manager nodes control the swarm, make scheduling decisions, and serve swarm API. Worker nodes run tasks assigned by managers.

Q4: How to initialize Docker Swarm?

Answer: Use 'docker swarm init' command. It creates a single-node swarm and makes current node a manager.

Q5: What is a Docker Service?

Answer: A Docker service is the definition of tasks to execute on manager or worker nodes. It's the central structure of swarm.

üöÄ CI/CD Pipelines

CI/CD Pipeline is an automated sequence of stages that code goes through from development to production.

Pipeline Types:

Declarative Pipeline

Structured, easier syntax

Scripted Pipeline

Groovy-based, more flexible

Sample Declarative Pipeline:

pipeline { agent any environment { DOCKER_REGISTRY = 'your-registry' IMAGE_NAME = 'myapp' } stages { stage('Checkout') { steps { git 'https://github.com/user/repo.git' } } stage('Build') { steps { sh 'mvn clean compile' } } stage('Test') { steps { sh 'mvn test' } post { always { junit 'target/test-results.xml' } } } stage('Package') { steps { sh 'mvn package' } } stage('Docker Build') { steps { script { def image = docker.build("${IMAGE_NAME}:${env.BUILD_ID}") } } } stage('Deploy') { steps { script { docker.image("${IMAGE_NAME}:${env.BUILD_ID}").run('-p 8080:8080') } } } } post { success { echo 'Pipeline succeeded!' } failure { echo 'Pipeline failed!' } } }

Creating Pipeline:

  1. New Item ‚Üí Pipeline
  2. Configure pipeline definition
  3. Choose Pipeline script or SCM
  4. Save and build

Pipeline Best Practices:

  • Use version control for Jenkinsfiles
  • Keep pipelines simple and modular
  • Use parallel stages for efficiency
  • Implement proper error handling
  • Include comprehensive testing

‚ùì Viva Questions - Unit VI

Q1: What is Jenkins?

Answer: Jenkins is an open-source automation server that enables Continuous Integration and Continuous Deployment through build automation and pipeline management.

Q2: Explain Jenkins Architecture.

Answer: Jenkins has Master-Agent architecture. Master controls pipelines and scheduling, while Agents execute build tasks. Jobs define build configurations.

Q3: What are Jenkins Nodes?

Answer: Jenkins nodes are machines (master or agents) that execute builds. Master node controls, agent nodes perform actual work.

Q4: What is Jenkins Pipeline?

Answer: Jenkins Pipeline is a suite of plugins supporting implementing and integrating continuous delivery pipelines as code.

Q5: Difference between Declarative and Scripted Pipeline?

Answer: Declarative uses structured syntax with predefined sections. Scripted uses Groovy language with more flexibility but complex syntax.

Q6: How does Jenkins integrate with Docker?

Answer: Jenkins can build Docker images, run containers, deploy to registries, and use Docker agents for builds through Docker plugins.
